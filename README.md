# On the Review of Agentic Refactoring Pull Requests: An Empirical Study

**Replication Package**



---

## Overview

This replication package contains the dataset, classification results, and analysis notebook for our empirical study on how human reviewers evaluate agentic refactoring pull requests. We analyzed 937 refactoring PRs generated by 5 AI coding agents (Devin, OpenAI Codex, GitHub Copilot, Cursor, and Claude Code) across 856 open-source GitHub repositories, focusing on human reviewer feedback and refactoring intent classification.

## Dataset Description

### `accepted_with_reviews.csv` / `rejected_with_reviews.csv`
Raw PR data. Each row is one pull request with the following columns:

| Column | Description |
|---|---|
| `id` | Unique PR identifier |
| `number` | PR number in the repository |
| `title` | PR title |
| `body` | PR description |
| `agent` | AI agent that created the PR (Devin, Copilot, OpenAI_Codex, Cursor, Claude_Code) |
| `user` | GitHub username of the agent account |
| `state` | PR state (closed) |
| `created_at` | Creation timestamp |
| `closed_at` | Closing timestamp |
| `merged_at` | Merge timestamp (null for rejected) |
| `repo_url` | Repository API URL |
| `html_url` | PR GitHub URL |

### `pr_review_comments.csv`
All review comments (human and bot) for PRs in the dataset.

| Column | Description |
|---|---|
| `id` | Comment identifier |
| `pr_id` | Links to `id` in the PR files |
| `user` | Commenter username |
| `user_type` | `User` or `Bot` |
| `body` | Comment text |
| `created_at` | Comment timestamp |

### `taxonomy_accepted_final.csv` / `taxonomy_rejected_final.csv`
RQ1 classification results using the AlOmar et al. (2022) Refactoring Review Taxonomy.

| Column | Description |
|---|---|
| `#` | Row index |
| `PR` | Short PR identifier (repo/pull/number) |
| `PR_Link` | Full GitHub URL |
| `Main_Category` | One or more taxonomy main categories (semicolon-separated) |
| `Subcategory` | Corresponding subcategories (semicolon-separated) |
| `Evidence` | Direct quote from reviewer comment supporting the classification |
| `Explanation` | Rationale for the classification |
| `Label` | `accepted` or `rejected` |
| `Status` | `Accepted` or `Rejected` |

## Taxonomy


| Category | Subcategories |
|---|---|
| **QUALITY** | Code smell, Internal quality attribute, External quality attribute, Technical debt, Design pattern, Coding convention, Lack of documentation |
| **REFACTORING** | Refactoring correctness, Behavior preservation, Refactoring co-occurrence with changes, Domain constraint |
| **OBJECTIVE** | Unclear goal, Unknown benefit, Potential side effect, Scope change, Feature support, Bug fixing |
| **TESTING** | Lack of coverage, Absence of result, Poor test quality |
| **INTEGRATION** | Configuration issue, Merge conflict, API management, Build failure |
| **MANAGEMENT** | No review activity, Forgotten review, Change dependency, Review guideline violation |

## How to Reproduce

### Requirements
```bash
pip install pandas anthropic scipy
```

### Steps

1. Clone this repository
2. Open `taxonomy_classification.ipynb` in Google Colab or Jupyter
3. Add your Anthropic API key in the designated cell
4. Run all cells in order

The notebook covers:
- **Step 1**: Load and verify the three CSV files
- **Step 2**: Filter bot comments and build reviewer lookup
- **Step 3**: Define the taxonomy classification prompt and Claude API call
- **Step 4**: Run classification on all PRs with independent human reviewer comments
- **Step 5**: Auto-tag PRs with no human reviewer comments as `MANAGEMENT / No review activity`
- **Step 6**: RQ1 analysis — category counts, subcategory breakdown, summary table
- **Step 7**: Export final classified CSVs

> **Note**: PRs with no independent human reviewer comments (i.e., only the PR author commented, or no comments at all) are automatically labeled `MANAGEMENT → No review activity` without calling the API.

---

## Classification Method

Reviewer comments were classified using the Claude Sonnet API (`claude-sonnet-4-5`) with a system prompt embedding the full taxonomy definitions. Each PR's independent human reviewer comments (excluding bot comments and PR author self-comments) were passed to the API. The model returns a JSON response with one or more classifications, each including the main category, subcategory, supporting evidence, and explanation. PRs with multiple distinct reviewer concerns receive multiple taxonomy labels.
